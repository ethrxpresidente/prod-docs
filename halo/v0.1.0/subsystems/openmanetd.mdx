# openmanetd

`openmanetd` is the OpenMANET manager daemon. It is a Go application that ties the mesh services together on a node: it drives `alfred` data exchange, consumes GNSS, and provides a multicast push-to-talk voice path. HALO builds `openmanetd` `1.2.0` from the OpenMANET source.

## Service

`openmanetd` runs under `procd` as a single instance. The init script starts it with an explicit config path and supervises it with a respawn policy.

```
PROG="/usr/bin/openmanetd"
CONF="/etc/openmanetd/config.yml"
```

The service registers two triggers. It reloads when the config file changes, and it restarts when a network interface comes up on `bat0` — so the daemon re-establishes its mesh bindings whenever the batman-adv interface is reconfigured. Stdout and stderr are routed to the system log.

Runtime dependencies pull in the mesh and audio stack: `kmod-batman-adv`, `alfred`, `gpsd`, the Opus and PortAudio libraries, ALSA, USB audio kernel modules, and `libevdev` for the push-to-talk input device.

## Configuration

Configuration is a YAML file at `/etc/openmanetd/config.yml`, installed from the package's sample config and preserved as a conffile across upgrades. The top-level keys configure logging, the mesh interface, and the on-disk database.

```yaml
logLevel: info
meshNetInterface: br-ahwlan
dbFile: /etc/openmanetd/openmanetd.db
```

### GNSS

The `gnss` block enables position handling and selects how position is republished. `openmanetd` can emit position as NMEA and as Cursor-on-Target, advertising itself as an external GNSS source.

```yaml
gnss:
  enable: true
  sendAsExternalGNSSSource:
    sendAsNMEA: true
    sendAsCoT: true
```

Position is sourced through `gpsd`, which HALO builds at `3.25` and which listens on port `2947` for clients requesting position, time, and velocity from an attached receiver.

### alfred

The `alfred` block configures `openmanetd` as an `alfred` participant. It runs as `primary` against the `bat0` batman-adv interface and the `alfred` control socket, and selects which data types it exchanges across the mesh.

```yaml
alfred:
  mode: primary
  batInterface: bat0
  socketPath: /var/run/alfred.sock
  dataTypes:
    gateway: true
    node: true
    position: true
    addressReservation: true
```

This is how gateway status, node descriptors, position, and address reservations propagate to every node in the mesh.

### Push-to-talk

`openmanetd` includes `pttgo`, a multicast push-to-talk voice application built on PortAudio, Opus, and `evdev`. It is disabled by default. When enabled, it joins a multicast group for voice traffic and listens for a key event from a USB audio or HID input device.

```yaml
ptt:
  enable: false
  mcastAddr: 224.0.0.1
  mcastPort: 5007
  pttKey: any
  loopback: true
  pttDevice: /dev/hidraw0/*
  pttDeviceName: Generic AB13X USB Audio
```
