# Boards and drivers

A HALO node needs a HaLow radio driver and a board support package that supplies firmware, defaults, and platform glue. This page covers the drivers and the three supported board families.

## Morse Micro HaLow driver

`kmod-morse` is the Morse Micro Wi-Fi HaLow driver, built from the Gateworks fork at version `1.16.4-gateworks`. It drives the MM6108-class chipset and auto-loads on boot. The driver supports SDIO, SPI, and USB attachment, selected at build time, and is configured with a default country of `US` and external crystal initialization.

The driver depends on the Morse firmware (`morse-fw`) and a board config (`morse-board-config`), plus the `mac80211` backport and MMC support. Optional build modes expose a debug build, monitor mode, vendor commands, and user-space access.

## Wi-Fi drivers

`wifi-openmanet` bundles the conventional Wi-Fi drivers an OpenMANET node may also need, so the access side and uplink can use standard radios alongside HaLow. It is built at `2025.11` and pulls in Intel `iwlwifi` (AX200/AX210), Qualcomm `ath11k` (WCN6855) and `ath10k` (QCA6174), and MediaTek `mt7915`/`mt7916`/`mt7921` drivers and firmware.

## Board support packages

Board support packages carry per-platform defaults, firmware, board.d files, and init scripts. They share a common base.

### bsp-common

`bsp-common` provides the common board support files installed on every OpenMANET board. The board-specific packages depend on it.

### bsp-bcm271x

`bsp-bcm271x` supports the Broadcom BCM271x platform — the Raspberry Pi reference node. It installs init scripts, board.d files, config, interface hotplug handlers, and Morse firmware. It depends on `bsp-common`, configfs, and USB gadget support. This is the platform behind the reference node: a BCM2711 with an MM6108 radio over SDIO.

### bsp-venice

`bsp-venice` supports the Gateworks Venice board. It adds board.d files, config, uci-defaults, and NTP configuration, and depends on `bsp-common`, `gpsd`, thermal and Gateworks hardware-monitor kernel modules, and `ntpd`. The `gpsd` and NTP dependencies reflect Venice's role as a GNSS-equipped node.

### bsp-halowlink2

`bsp-halowlink2` supports the Morse Micro HaLow Link 2 (`morse,artini` / MT7621). It sets a default channel of `28` (`916 MHz`, `8 MHz` bandwidth) and selects the board's BCF, applies a SAE mesh fix (`ieee80211w=0` and `sae_pwe=0`), and ships a custom BCF. The chip firmware itself is supplied separately by `morse-fw-8108`.

## Firmware and BCF

Board packages install Morse firmware and board configuration files into `/lib/firmware/morse/`. The BCF — board configuration file — tunes the radio for the specific board layout and antenna. Each board selects its own BCF; the HaLow Link 2, for example, uses an extended BCF for its hardware.
